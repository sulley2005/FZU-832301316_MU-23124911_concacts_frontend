# 代码风格规范

## HTML 规范

1. 使用语义化标签，如 `<header>`, `<nav>`, `<main>`, `<footer>` 等
2. 缩进使用 4 个空格
3. 属性值使用双引号 `"` 包裹
4. 标签名小写，属性名小写
5. 自闭合标签必须带 `/`，如 `<img />`
6. 合理使用注释，解释复杂结构

## CSS 规范

1. 使用类选择器为主，避免使用 ID 选择器
2. 采用 BEM 命名规范（block__element--modifier）
3. 样式按功能模块组织，使用注释分隔
4. 缩进使用 4 个空格
5. 属性声明顺序：布局相关 > 盒模型相关 > 文本相关 > 其他
6. 避免使用 `!important`

## JavaScript 规范

1. 变量命名使用 camelCase 风格
2. 常量命名使用 UPPER_SNAKE_CASE 风格
3. 函数命名使用 camelCase 风格，且动词开头
4. 缩进使用 4 个空格
5. 使用 `const` 和 `let` 声明变量，避免使用 `var`
6. 使用箭头函数简化匿名函数
7. 使用模板字符串代替字符串拼接
8. 使用 `===` 和 `!==` 进行比较
9. 每个函数功能单一，避免过长函数
10. 使用注释解释复杂逻辑

## 通用规范

1. 代码提交前进行格式化
2. 避免冗余代码和重复逻辑
3. 注重代码可读性和可维护性
4. 遵循 DRY (Don't Repeat Yourself) 原则
